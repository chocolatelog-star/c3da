# 动态多三元组训练与增强设计

## 1. 背景与目标

当前 `rest16 -> laptop14` 最佳实验的原始微平均 F1 为 48.93%。该实验在原有每句最多保留一个三元组的高精度伪标签上，额外加入了 73 条完整双三元组伪标签，证明完整结构监督能够明显改善多三元组抽取。

但是，现有流程仍包含每句最多保留 1、2 或 3 个三元组的硬限制，并且“完整双三元组”是专门分支，不能自然处理 3 个及更多三元组。目标是把流程改为：

1. 源域按人工标注中的真实三元组集合训练，不裁剪标签。
2. 专门加强提取器对多三元组句子的覆盖能力。
3. 目标域伪标签逐个三元组进行质量判断，不按整句数量拒绝。
4. 数据增强编辑一个三元组时，保持该句其余三元组及其对应关系不变。
5. 不使用目标域隐藏金标训练、筛选伪标签或选择检查点。
6. 不额外训练固定的三元组数量预测器。

## 2. 数据依据

源域训练集共 857 句，人工标注数量分布如下：

| 每句三元组数 | 句子数 | 比例 |
|---:|---:|---:|
| 1 | 505 | 58.93% |
| 2 | 229 | 26.72% |
| 3 | 73 | 8.52% |
| 4 个及以上 | 50 | 5.83% |

源域共有 352 句多三元组样本，占 41.07%；其中 123 句至少包含 3 个三元组，占 14.35%。目标域训练集隐藏金标中的对应比例约为 39.62% 和 14.46%。因此，多三元组不是少量异常情况，而是需要独立优化的主要结构类型。

## 3. 方案边界

本轮采用“动态结构质量筛选”，不采用以下方案：

- 不把伪标签上限从 2 简单改为 3。此前直接放宽数量已经引入低质量裁剪样本。
- 不使用目标域隐藏金标确定一条句子应包含多少三元组。
- 不新增独立数量分类器，避免数量预测误差进一步约束生成结果。
- 不改成逐轮抽取一个三元组的迭代式架构，避免扩大本轮变量和累积误差。

## 4. 提取器多三元组训练

### 4.1 完整标签监督

源域每条人工标注样本始终使用完整、规范化后的标签集合。三元组使用确定性顺序输出，避免同一集合因排列不同产生额外序列损失。

### 4.2 结构平衡权重

只对源域人工标注样本增加温和的结构权重：

| 三元组数 | 建议初始权重 |
|---:|---:|
| 1 | 1.00 |
| 2 | 1.15 |
| 3 | 1.25 |
| 4 个及以上 | 1.30 |

权重必须可由命令行配置，并在训练日志中输出各数量分组的样本数、平均权重和有效损失。目标域伪标签和增强数据不直接继承该人工标注权重，避免不可靠多三元组被额外放大。

### 4.3 检查点诊断

每个评估轮次保留验证损失，同时生成以下源域验证集指标：

- 总体原始微平均 F1。
- 单三元组原始微平均 F1。
- 双三元组原始微平均 F1。
- 三三元组原始微平均 F1。
- 4 个及以上三元组原始微平均 F1。
- 精确数量比例、少生成比例和多生成比例。

检查点选择以总体原始微平均 F1 为主，在总体指标接近时优先多三元组 F1。目标域隐藏金标只用于实验完成后的分析。

## 5. 动态伪标签筛选

### 5.1 移除行级数量上限

伪标签筛选不再因为一行预测超过固定数量而拒绝整行。旧参数保留兼容读取能力，但动态模式不得执行 `too_many_triplets` 拒绝。

### 5.2 逐三元组过滤

每个预测三元组分别执行：

1. 情感类别必须属于正向、负向或中性。
2. 方面词和观点词必须能在原句中定位。
3. 排除空词项、噪声词项和非法边界。
4. 计算方面词与观点词距离，并应用同一套可配置距离规则。
5. 保留通过检查的三元组，删除失败三元组。

如果预测4个三元组且3个通过，则保留包含这3个三元组的伪标签；不因为原始数量或最终数量为3而拒绝。

### 5.3 动态行权重

保留一行伪标签的前提是至少有一个三元组通过。样本权重由以下因素共同决定：

- 原始伪标签基础置信度。
- 过滤后保留三元组数与过滤前数量的比例。
- 是否发生词项修正。
- 全部保留三元组是否能在原句定位。

第一版使用保守的行级权重，不实施逐输出词元加权。权重上限不得超过现有高精度伪标签权重0.65；发生部分删除的行应降低权重。分析文件必须记录每行过滤前后数量、删除原因、最终权重和来源类型。

### 5.4 防止数量放宽引入假阳性

动态模式必须输出按预测数量分组的伪标签统计，并设置质量退化保护：

- 统计1、2、3、4个及以上三元组伪标签数量。
- 训练前只使用词项覆盖率、过滤保留率、平均置信度、部分删除率和数量分布等无标签质量代理进行退化保护。
- 在完整实验结束后的隐藏金标分析模式下分别报告各组精确率、召回率和 F1；这些指标不得用于调整阈值、筛选伪标签、选择检查点或决定样本是否进入训练。

## 6. 多三元组结构保持增强

### 6.1 修复训练与推理不一致

当前互相掩码生成器训练逐个三元组创建编辑样本，但训练标签只保存被修改的单个三元组；增强推理却把整句所有三元组作为目标标签。本轮统一为完整集合监督。

示例：

```text
原标签：T1 + T2 + T3
编辑目标：T2 -> T2'
新标签：T1 + T2' + T3
```

方面词通道与观点词-情感通道都必须保存完整的新标签集合。生成文本只编辑选中的词项，其余词项和对应三元组保持不变。

### 6.2 结构保持增强

优先对源域人工标注和高质量目标域伪标签中的真实多三元组句子执行结构保持增强：

- 每个增强请求只编辑一个三元组，减少多个局部修改造成的语义冲突。
- 可对同一句创建不同编辑位置的候选，但最终选择阶段限制同源句占比。
- 请求元数据保存完整原标签、被编辑索引、旧三元组、新三元组及未修改三元组。
- 生成器训练数据覆盖1、2、3、4个及以上三元组分组。

### 6.3 动态结构组合增强

结构组合属于第二阶段变量，不与结构保持增强同时首次引入。组合数量按照源域人工标签数量分布采样，不固定为2或3；实际可生成数量受可用兼容标签和最大输出长度限制，而不是筛选硬上限。

组合标签必须满足：

- 方面词不重复且边界有效。
- 观点词和情感组合合法。
- 标签之间通过现有领域相似度与兼容性检查。
- 生成后接受完整集合回抽验证。

## 7. 增强质量过滤

多三元组增强样本依次执行：

1. 文本质量和领域提示泄漏检查。
2. 所有请求方面词、观点词的文本覆盖检查。
3. 自然语言推理一致性检查。
4. 提取器回抽检查。
5. 完整集合覆盖检查。

回抽结果分为：完整覆盖、部分覆盖、额外生成、完全失败。只有完整覆盖进入严格增强主实验；其他结果保留在诊断文件，不进入训练。不能因为其中一个三元组正确就接受整条多三元组增强样本。

## 8. 实验隔离

实验按照以下顺序进行，每一步使用独立输出标签、结果文件和阶段状态：

| 实验 | 训练组成 | 目的 |
|---|---|---|
| A | 当前48.93最佳流程 | 对照组 |
| B | 多三元组提取器 + 动态伪标签 + 旧增强 | 验证提取器和动态伪标签 |
| C | B + 多三元组结构保持增强 | 验证完整集合编辑训练 |
| D | C + 少量动态结构组合增强 | 验证新增多结构生成 |

B因提取器训练策略改变，必须从本地 `J:\nlp\models\t5-base-py` 重新训练提取器。C和D可复用B的提取器及动态伪标签，但生成器或增强逻辑改变时必须重新生成对应训练文件和增强文件。

## 9. 资源、续跑与产物

- 所有训练使用 RTX 3070 8GB 显存配置。
- 保持单卡、半精度、梯度检查点和梯度累积策略。
- 每个训练与生成阶段必须显示进度条。
- 提取器、生成器和最终模型必须定期保存检查点并支持自动断点续跑。
- 阶段状态必须包含代码版本、命令参数、输入文件摘要和产物路径，禁止错误复用旧阶段。
- 新实验不得覆盖当前48.93最佳模型及其结果。

## 10. 测试与验收

### 10.1 单元测试

- 动态筛选能够保留3个及更多合法三元组。
- 一行中部分三元组失败时只删除失败项。
- 动态模式不触发数量过多拒绝。
- 多三元组掩码训练标签保留未编辑三元组。
- 方面词与观点词-情感两个通道均保持完整集合。
- 完整覆盖过滤拒绝部分覆盖的增强样本。
- 旧固定数量模式和旧实验文件仍可读取，不改变历史产物。

### 10.2 代码验证

- 全部相关单元测试通过。
- Python 语法检查通过。
- 干运行命令展示独立阶段名、独立输出标签和正确的断点续跑参数。
- 工作区无未记录代码改动。

### 10.3 实验成功标准

第一阶段先运行 `rest16 -> laptop14` 单方向。主要比较原始微平均 F1，并同时要求：

- 总体原始微平均 F1 不低于48.93%。
- 多三元组原始 F1 高于46.13%，或多三元组召回率有明确提升且总体精确率未明显下降。
- 3个及以上三元组分组的召回率和精确数量比例得到改善。
- 动态伪标签精确率没有因取消数量上限而明显下降。

单方向通过后，再执行3个随机种子复验；稳定后扩展到其余跨域方向。

## 11. 风险与保护措施

| 风险 | 保护措施 |
|---|---|
| 多三元组加权导致过度生成 | 使用温和且有上限的源域权重，监控多生成比例 |
| 取消数量上限引入假阳性 | 逐三元组过滤、动态降权、按数量分组诊断 |
| 增强编辑破坏未修改三元组 | 完整标签训练与完整集合回抽过滤 |
| 多变量同时改变后无法归因 | 按 A、B、C、D 顺序隔离实验 |
| 8GB 显存不足 | 保持批量大小1、梯度累积、半精度与梯度检查点 |
| 中断后重复长时间训练 | 每阶段保存检查点并由阶段状态执行自动续跑 |
