# 编码器上下文方面词-观点词配对损失设计

## 1. 目标

第五项实验直接优化 ASTE（方面情感三元组抽取）中的方面词-观点词配对。当前最佳模型的主要问题是召回率、多三元组拆分和方面词-观点词对应关系不足；第四项证明只提高增强文本的标签一致性不能解决该问题。

本次只在当前最佳最终 5 轮训练中增加编码器上下文配对损失。训练数据、提取器、421 条 hp1 高精度伪标签、150 条旧版标签到文本增强、DANN（领域对抗神经网络）和情感原型对比学习均保持不变。

主验收指标是 raw F1（原始 F1）是否超过 46.82；同时重点检查多三元组句子的召回率和配对错误。

## 2. 表示与有效样本

对输入句子经过 T5 encoder（T5 编码器）得到上下文表示 `H`。对每个金标三元组 `(aspect, opinion, sentiment)`，在输入句子中定位方面词和观点词跨度并平均池化：

```text
a_i = mean(H[aspect_span_i])
o_i = mean(H[opinion_span_i])
```

只使用满足以下条件的样本：

- `source_gold`（源域人工标注）。
- 同一句至少包含 2 个有效三元组。
- 方面词和观点词都能在编码器输入中定位。
- 最多使用前 4 个有效三元组，保持现有显存边界。

目标伪标签和增强样本不参与配对损失，避免错误配对被强化。批大小仍为 1；负例来自同一句中的其他三元组，因此不依赖跨批次负例。

## 3. 多正例双向损失

相似度矩阵为：

```text
S_ij = cosine(a_i, o_j) / temperature
```

默认温度为 0.1。不能简单把矩阵对角线设为唯一正例，因为真实数据可能出现：

- 同一个方面词对应多个观点词。
- 同一个观点词同时描述多个方面词。

方面词到观点词方向中，与当前方面词跨度相同的所有已标注观点词都是正例；观点词到方面词方向中，与当前观点词跨度相同的所有已标注方面词都是正例。每个方向使用多正例 log-sum-exp（对数指数和）损失：

```text
L_i = -log(sum(exp(S_i,p) for p in positives) / sum(exp(S_i,j) for j in candidates))
```

总损失为两个方向的平均：

```text
L_pair = 0.5 * (L_aspect_to_opinion + L_opinion_to_aspect)
L_total = L_generation + 0.03 * L_DANN + 0.01 * L_sentiment + 0.01 * L_pair
```

若某个方向没有真实负例，该方向不计入损失；若整句没有有效正负对比，配对损失为 0。

## 4. 为什么不启用现有解码器配对损失

现有实现从线性化目标标签定位跨度，再在 decoder hidden state（解码器隐藏状态）上计算单向对角交叉熵，存在三个问题：

1. T5 解码器输入右移，标签跨度与隐藏状态位置有错位风险。
2. 标签序列表示不等于原始句子的上下文表示。
3. 对角线唯一正例会把共享方面词或共享观点词的真实关系当成负例。

新实现保留旧函数以兼容历史代码，但主实验改用编码器输入跨度和多正例双向损失。

## 5. 参数、日志与输出隔离

新增或明确以下参数：

```text
--lambda_pairing_loss 0.01
--pairing_temperature 0.1
--pairing_source_only
--max_pairing_triplets 4
--min_pairing_triplets 2
```

训练日志输出：

- `pairing_loss`（配对损失）。
- `pairing_aspect_accuracy`（方面词到观点词多正例准确率）。
- `pairing_opinion_accuracy`（观点词到方面词多正例准确率）。
- `pairing_active_rows`（有效多三元组句数）。
- `pairing_active_pairs`（有效配对数）。

新模型、预测、指标和汇总统一使用独立标签：

```text
pairing_encoder_l001_source_only
```

不得覆盖当前最佳模型或默认汇总文件。

## 6. 严格单变量实验

实验继续使用：

```text
runs\bgca_aste_stage1_domain_prompt_text_v1\rest16_to_laptop14
```

复用已存在的当前最佳训练文件：

```text
final_train_strict_aug150_w020_label_to_text_gen.jsonl
final_dev_strict_aug150_w020_label_to_text_gen.jsonl
```

批处理脚本应跳过准备、提取器、伪标签、生成器和增强阶段，只训练带独立标签的最终模型并评估。汇总文件也必须带配对实验标签。

训练参数保持 RTX 3070 8GB 配置：训练批大小 1、验证批大小 2、梯度累积 16、FP16（半精度浮点）、梯度检查点、5 轮、学习率 3e-4、随机种子 1000，并支持 `--resume_from_checkpoint auto`（自动从检查点恢复）。

## 7. 测试与验收

实现前先添加失败测试，覆盖：

1. 从编码器输入正确定位多词方面词和观点词。
2. 单三元组、伪标签和增强样本不产生配对特征。
3. 正确配对的损失低于交换配对。
4. 重复方面词或重复观点词使用多正例，不把真实关系当负例。
5. 无有效负例时返回有限的 0，而不是 NaN（非数值）。
6. 日志输出损失、两个方向准确率和有效计数。
7. 完整 dry-run（仅打印不执行）只包含最终训练和评估，模型与汇总标签独立。
8. 旧 `lambda_pairing_loss=0`（配对损失关闭）路径行为不变。

完整实验结束后比较总体 raw/fixed（原始/修正）指标、分情感指标、多三元组子集指标和配对错误。若 raw F1 低于 46.82，先分析再征得许可删除大型模型。
